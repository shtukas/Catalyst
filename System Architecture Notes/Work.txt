
WorkItem (CoreDataTx) {
    uuid                : String         # _objectId_
    schema              : "workitem"     # _schema_
    unixtime            : Float          # _unixtime_
    description         : String         # _description_

    "workItemType"      : String         # _payload1_ # "General" | "PR" | "RotaItem"
    "trelloLink"        : String or null # _payload2_
    "prLink"            : String or null # _payload3_
    "gitBranch"         : String or null # _payload4_
}

At [creation] or at [description update] the directory filename is the date together with the sanitised version of the description, but at any time the directory name can be manually updated because the resolution of the directory name is done using a marker in the folder itself.

The uuid is written inside the directory in a .NxC144FB7A file.
